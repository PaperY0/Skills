# Ability Tree — 能力树生成规则

PRES Layer 2。根据项目画像动态生成能力地图，替代固定课程。

---

## 生成规则

### 第一步：读取项目画像

从 Project Analyzer 输出的技术栈 + 重要度 ⭐。

### 第二步：映射为能力节点

| 技术栈分类 | 能力节点 |
|-----------|---------|
| 前端框架 (React/Vue/Angular) | 能够识别组件 / 能够理解 JSX/模板 / 能够理解 Hooks/生命周期 / 能够管理状态 |
| TypeScript | 能够读懂类型定义 / 能够使用接口约束数据 / 能够利用类型安全重构 |
| 数据库/ORM | 能够识别 Repository / 能够画数据流 / 能够设计表结构 / 能够写查询 |
| 状态管理 | 能够识别 Store / 能够理解全局状态 / 能够选择合适的状态方案 |
| AI 服务 | 能够设计 Prompt / 能够设计 API 路由 / 能够做 Zod 校验 / 能够做容错重试 |
| 后端框架 (Express/Nest/FastAPI) | 能够理解路由 / 能够设计 API / 能够理解中间件 |
| CSS/UI | 能够使用组件库 / 能够理解响应式 / 能够定制主题 |
| 构建工具 (Vite/Webpack) | 能够理解构建流程 / 能够配置开发环境 |

### 第三步：按 ⭐ 排序生成树

```
第一阶段（固定）：系统认知
  能够看懂目录
  能够找到入口
  能够理解模块职责
  能够画出数据流

后续阶段（动态）：按重要度降序
  ⭐⭐⭐⭐⭐ → 最先教
  ⭐⭐⭐⭐   → 第二批
  ⭐⭐⭐     → 第三批
```

---

## 示例：Lumi 项目能力树

```
系统认知
├── 能够看懂目录结构
├── 能够找到项目入口
├── 能够理解模块职责
└── 能够画出数据流

React (⭐⭐⭐⭐⭐)
├── 能够识别组件
├── 能够理解 JSX
├── 能够理解 Hooks (useState/useEffect)
└── 能够理解组件通信

TypeScript (⭐⭐⭐⭐⭐)
├── 能够读懂 interface/type
├── 能够理解类型约束的价值
└── 能够利用类型安全防 Bug

AI 系统 (⭐⭐⭐⭐⭐)
├── 能够理解为什么 AI 放后端
├── 能够设计 Prompt
├── 能够理解 JSON Schema 校验
└── 能够理解容错和重试

数据层 (⭐⭐⭐⭐)
├── 能够识别 Repository 模式
├── 能够理解 IndexedDB vs localStorage
└── 能够画出数据存取流程

状态管理 (⭐⭐⭐⭐)
├── 能够识别 Zustand Store
├── 能够理解全局状态 vs 本地状态
└── 能够选择合适的状态方案

后端 (⭐⭐⭐⭐)
├── 能够理解 Express 路由
├── 能够理解 API 请求全链路
└── 能够理解前后端通信
```
