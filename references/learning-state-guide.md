# 学习状态管理指南

PRE 不只是教课。它管理用户从零到独立开发的完整学习状态。

---

## 学习状态文件

当用户开始学习一个项目时，在 skill workspace 下创建项目专属的学习状态目录：

```
<skill-workspace>/<project-name>/
├── MISSION.md              ← 学习目标（为什么要学）
├── NOTES.md                ← 用户偏好 + 教学笔记
├── COURSE-PLAN.md          ← 8 阶段完整课程计划
└── LEARNING-RECORDS/       ← 每课后的学习记录
    ├── 0001-lesson-01.md
    ├── 0002-lesson-02.md
    └── ...
```

---

## MISSION.md 模板

```markdown
---
name: learn-fullstack-via-<project>
description: <一句话学习目标>
status: active
---

## 学习目标

我要通过 <项目名> 项目，学会自己能独立设计、开发、部署一个完整产品。

## 当前水平

- 编程经验：<零基础/了解 HTML CSS/能写简单页面/...>
- 想做但还不会：<具体能力>
- 最终目标：<看懂代码/能改代码/能自己设计系统>

## 学习风格

- [x] 要知道"为什么"，不是"怎么做"
- [x] 喜欢生活类比
- [x] 中文教学
- [ ] 喜欢快节奏
- [ ] 喜欢详细展开各项
```

### 为什么需要 MISSION.md？

- 每一课都对准学习目标，不会偏离
- 用户目标变了，MISSION.md 更新，课程方向跟着变
- 没有目标的教就像没有目的地导航——永远到不了

## 最近发展区计算

### 如何判断用户当前水平

读取 LEARNING-RECORDS 中最近的记录：
1. 用户理解了哪些概念？（✅标记的）
2. 用户卡在哪里？（❌和⚠️标记的）
3. 用户下一个需要的是什么能力？

### 选择下一课内容

```
用户已掌握的 ────→ 最近发展区（下一课）────→ 还不能学的
   ✅✅✅               🎯               ❌❌❌
```

下一课必须是"用户差一点点就能够到"的——不是已经会的，也不是完全够不到的。

## 检索练习（Retrieval Practice）

### 时机

- 每节新课的开头
- 第 5 课、第 10 课、第 15 课做跨课综合复习

### 问题设计

每节课的⑬步骤生成 3 个回顾问题：

1. **概念回忆题**：用自己的话解释上一课的核心概念
2. **生活类比题**：用日常场景重新解释那个概念
3. **后果推理题**：如果删掉那个模块，项目会发生什么

### 反馈标准

| 回答质量 | 判定 | 下一步 |
|---------|------|--------|
| 能用自己的话准确解释 | ✅ 已掌握 | 可以学新课 |
| 大致对，但细节模糊 | ⚠️ 部分掌握 | 快速回顾核心点再继续 |
| 理解有误或说不出来 | ❌ 未掌握 | 重新教这一课，不能继续 |

---

## 间隔复习（Spaced Review）

每 5 节课后，插入一次跨课复习：
- 把前 5 课的核心概念串成一个数据流故事
- 让用户画出完整的架构图（从记忆里，不看着画）
- 标注哪些概念之间有关联（例如 Repository 依赖 Database，Store 依赖 Repository）
