# PRES 13+1 步课程模板（含 7 层认知 + 代码雷达）

每课必须同时生成 `.md` + `.html` 两个文件。

---

```
Lesson XX：<课程标题>

━━━━━━━━━━━━━━━━━━━━━━━━━━━━

① 今天只解决一个问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

一句话，以问题形式。例如：
"为什么页面不能直接操作数据库？"
"为什么页面A改了数据，页面B能看到？"

❌ 禁止："今天学 Repository"


② 为什么会有这个问题？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

从真实场景出发，不是代码角度。

示例："你在淘宝买东西，'我的订单'和'待评价'都能看到同一个订单数据。这是怎么做到的？"

如果不用正确方法会怎样？列出 3-5 个具体后果（生活例子解释）。


③ 企业为什么这样设计？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

历史背景 + 小公司做法 vs 大公司做法 + 什么时候该升级。


④ 现实世界类比
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

完整日常故事（快递站/餐厅/超市/银行/微信群……），必须包含技术出现前 vs 出现后的对比。


⑤ 它在项目哪个文件？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

具体文件路径 + 数据流中的位置（ASCII图标注）。


⑥ 打开这些文件
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

列出 2-4 个要看的文件，每个一句话说明。


⑦ 七层认知：代码认知（⭐核心）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

必须包含 6 个子步骤：

① 最小可运行样例（Mini Example）
  3-5 行代码。去掉所有抽象、泛型、错误处理。只保留核心。
  先给"没有它"的版本，再给"有它"的版本。

② 本体长什么样？
  最小版本的这个概念是什么结构。例如最小 Repository 类。

③ 调用流程
  ASCII 箭头图 + 对应代码注释。

④ 左右对比代码
  左侧：没有 XX 的代码（到处散落 db.xxx）
  右侧：有 XX 的代码（统一通过 XX 调用）
  一眼看出区别。

⑤ 去项目里找它
  真正打开项目的对应文件。把企业版代码和最小版放一起对比。
  "其实一样，只是多了一些容错。"

⑥ 企业为什么写成这样？
  为什么用 findById 而不是 get？因为企业规范，不是必须。


⑧ 如果删掉它会怎样？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

反面教材：删掉后项目出现什么具体问题。放错误代码对比。


⑨ 企业最佳实践
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

命名规范 / 文件组织 / 错误处理 / 常见坑 + 解决方法。


⑩ 验收
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

4 个问题确认真正理解：
- 用自己的话解释它是什么
- 它在项目哪个文件里
- 删掉会发生什么
- 什么时候需要它，什么时候不需要


⑩.5 知识检测（交互式测验）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

嵌入 quiz 组件（引用 quiz.css + quiz.js）。
- 4 道场景判断题（禁止"背定义"题）
- 每题 4 选项，正确+错误选项都有解释
- 即时反馈 + 总分评级（<50%重学 / ≥50%鼓励 / 100%完美）


⑪ 能力成长（认知成长系统）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  知道是什么      ✅ / ❌
  知道为什么      ✅ / ❌
  会修改         ✅ / ❌
  会自己写       ✅ / ❌
  会自己设计     ✅ / ❌


⑫ 技术栈百科
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

关联技术拓展 + 面试方向 + 高质量参考资源链接。


⑬ 检索练习
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

3 个回顾问题（下一课开始时问用户）。


⑭ 代码雷达（⭐新增）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

以后看到这些特征，立即知道这是 XX：

  class XXXRepository
  findById() / save() / update() / delete()
  db.xxx.get() / db.xxx.put()

目标：不是"我告诉你"，而是"你的眼睛能自己识别"。
```
