## 课程输出格式规则

| 格式 | 文件名 | 何时生成 |
|------|--------|---------|
| **Markdown** | `lessons/L<NN>-<slug>.md` | 每课必须 |
| **HTML** | `lessons/L<NN>-<slug>.html` | 每课必须 |

两种格式包含相同内容，HTML 版本负责交互展示（可折叠段落、语法高亮、导航栏），Markdown 版本负责可编辑、可 Git 跟踪。

## 使用示例：Lesson 04 — Repository 模式

### ① 今天只解决一个问题
为什么页面不能直接操作数据库？中间需要一个"仓库管理员"。

### ② 为什么会有这个问题？
你开了一家快递站。100个顾客来取快递。你不能让100个顾客自己进仓库翻——你会请一个管理员。顾客说"取张三的快递"，管理员进去找。写代码同理——100个页面不能各自直接翻数据库。

### ③ 企业为什么这样设计？
最早 Web 开发把 SQL 写在页面里。3个页面没毛病。99个页面 → 改一个字段名改99处 → 发明 Repository 把数据操作集中管理。

### ④ 现实世界类比
你去银行取钱。不是你自己进金库拿。是告诉柜员 → 柜员进金库取 → 登记 → 给你。Repository 就是这个柜员。

### ⑤ 它在项目哪个文件？
`src/lib/db/repositories/userProfileRepo.ts`
